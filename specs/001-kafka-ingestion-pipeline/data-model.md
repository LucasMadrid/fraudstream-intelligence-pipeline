**Branch**: `001-kafka-ingestion-pipeline` | **Date**: 2026-03-30

## Entities

### 1. `TransactionEvent` (API channel) — Kafka record on `txn.api`

The canonical event written to the `txn.api` topic. Immutable after publication. Corresponds directly to the Avro schema `txn-api-v1.avsc`.

| Field | Type | Nullable | Source | Notes |
|---|---|---|---|---|
| `transaction_id` | `string` (UUID v4) | No | Caller or generated at producer | Partition key + dedup key downstream |
| `account_id` | `string` | No | Caller | Partition key for stateful aggregations |
| `merchant_id` | `string` | No | Caller | |
| `amount` | `bytes` (decimal, p=18, s=4) | No | Caller | Never `double`; two's-complement big-endian |
| `currency` | `string` (ISO-4217) | No | Caller | e.g. `"USD"`, `"EUR"` |
| `event_time` | `long` (epoch_ms) | No | Caller | Milliseconds since Unix epoch |
| `processing_time` | `long` (epoch_ms) | No | Producer | Set by producer at receipt |
| `channel` | `string` | No | Producer constant `"API"` | Valid values: POS, WEB, MOBILE, API |
| `card_bin` | `string` (6 chars) | No | PIIMasker | First 6 digits of PAN |
| `card_last4` | `string` (4 chars) | No | PIIMasker | Last 4 digits of PAN |
| `caller_ip_subnet` | `string` | No | PIIMasker | IPv4 /24 or IPv6 /64 truncated |
| `api_key_id` | `string` | No | Caller | Identifies the API credential used |
| `oauth_scope` | `string` | No | Caller | OAuth scope string; not validated by producer |
| `geo_lat` | `float` | Yes (`["null","float"]`) | Caller | May be absent for API clients |
| `geo_lon` | `float` | Yes (`["null","float"]`) | Caller | May be absent for API clients |
| `masking_lib_version` | `string` | No | PIIMasker | e.g. `"1.0.0"`; enables skew detection |
| `schema_version` | `string` | No | Producer constant `"1"` | Incremented on topic-level schema change |

**Partition key:** `account_id` (hashed). Ensures all transactions for the same account land on the same partition, enabling downstream stateful velocity aggregations without shuffling.

**Retention:** Raw events retained indefinitely in `txn.api` (Kafka log compaction disabled; time-based retention 7 years per regulatory requirement set at broker/topic level).

### 2. `DLQEnvelope` — Kafka record on `txn.api.dlq`

Written when an event cannot be published to `txn.api`. Contains the original payload and error context for later inspection and optional replay.

| Field | Type | Nullable | Notes |
|---|---|---|---|
| `dlq_id` | `string` (UUID v4) | No | Generated by DLQ producer |
| `source_topic` | `string` | No | Always `"txn.api"` |
| `original_payload` | `string` | No | JSON-serialised original (pre-masking) payload; may contain raw PII — treat as sensitive |
| `error_type` | `string` | No | e.g. `"SerializationError"`, `"BrokerError"` |
| `error_message` | `string` | No | Full exception message |
| `failed_at` | `long` (epoch_ms) | No | When failure occurred |
| `producer_host` | `string` | No | Hostname of the producer that failed |
| `masking_applied` | `boolean` | No | `true` if PII masking ran before failure; `false` if failure was pre-masking |

**Retention:** 7 days (configurable). DLQ is for operational triage, not long-term storage.

> **Security note:** `original_payload` may contain unmasked PII if the failure occurred before masking ran (`masking_applied=false`). The `txn.api.dlq` topic must have equivalent access controls to the most sensitive upstream source.

### 3. `RawPAN` — in-process only, never serialised

| Field | Type | Notes |
|---|---|---|
| `_digits` | `str` | Full digit string; `__slots__` only; never exposed via `__repr__` or `__str__` |

**Lifecycle:** Created at the HTTP/gRPC ingestion boundary from caller-supplied card number. `.mask()` is called immediately; the `RawPAN` object is discarded. Never stored in any field, dict, or log.

**Leak prevention:** `__repr__` → `"RawPAN(***)"`. `__reduce__` → raises `TypeError` (blocks pickle). Not exported from the `pii_masking` package's public API.

---

### 4. `MaskedPAN` — passed to serialiser

| Field | Type | Notes |
|---|---|---|
| `bin6` | `str` | First 6 digits |
| `last4` | `str` | Last 4 digits |
| `length` | `int` | Original PAN length (15 or 16) |

Frozen dataclass. Maps to `card_bin` + `card_last4` in `TransactionEvent`.

---

### 5. `MaskingConfig` — producer configuration

| Field | Type | Default | Notes |
|---|---|---|---|
| `ipv4_prefix` | `int` | `24` | CIDR prefix for IPv4 truncation |
| `ipv6_prefix` | `int` | `64` | CIDR prefix for IPv6 truncation |
| `reject_invalid_pan` | `bool` | `True` | If `True`, invalid PANs raise `InvalidPANError` |

Frozen dataclass injected into pure masking functions. One instance per producer process; no runtime mutation.

## State Transitions

### TransactionEvent lifecycle

```
[Caller HTTP/gRPC request]
        │
        ▼
[ProducerService.receive()]
   - Deserialise request body
   - Validate required fields (FR-001, FR-002)
        │
        ├── validation error ──► [DLQ: txn.api.dlq] (error_type=ValidationError)
        │
        ▼
[PIIMasker.mask()]
   - RawPAN → MaskedPAN (Luhn check, BIN+last4)
   - IP → truncated subnet
   - masking_lib_version stamped
        │
        ├── masking error ──► [DLQ: txn.api.dlq] (masking_applied=false)
        │
        ▼
[AvroSerializer.serialize()]
   - Validate against Schema Registry schema
   - Produce Confluent wire format (magic + schema_id + avro_payload)
        │
        ├── schema error ──► [DLQ: txn.api.dlq] (error_type=SerializationError)
        │
        ▼
[KafkaProducer.produce()]
   - Partition by account_id hash
   - Idempotent (enable.idempotence=True)
   - acks=all
        │
        ├── broker error (retries exhausted) ──► delivery_callback ──► [DLQ: txn.api.dlq]
        │
        ▼
[txn.api topic] ── downstream Flink enrichment / scoring pipeline
```

## Validation Rules

| Field | Rule |
|---|---|
| `transaction_id` | Must be a valid UUID v4 format |
| `amount` | Must be > 0 and ≤ 99,999,999,999,999.9999 |
| `currency` | Must be a 3-character ISO-4217 code |
| `event_time` | Must be within ±5 minutes of producer `processing_time` (clock skew guard) |
| `channel` | Must be one of: `POS`, `WEB`, `MOBILE`, `API` |
| `card_number` (input, pre-mask) | Must pass Luhn check and be 15 or 16 digits after normalisation |
| `caller_ip` (input, pre-mask) | Must be a valid IPv4 or IPv6 address; multicast raises `ValueError` |
| `api_key_id` | Must be non-empty string |
| `oauth_scope` | Must be non-empty string |

## Kafka Topic Configuration

### `txn.api`

| Config | Value | Rationale |
|---|---|---|
| `partitions` | 12 | Parallelism for downstream Flink; divisible by common replication factors |
| `replication.factor` | 3 | ISR durability for `acks=all` |
| `min.insync.replicas` | 2 | Ensures at least 2 replicas before ack |
| `retention.ms` | 220,752,000,000 (~7 years) | Regulatory minimum |
| `cleanup.policy` | `delete` | No compaction; immutable event log |
| `message.max.bytes` | 1,048,576 (1 MB) | Transaction events are small; 1 MB is generous headroom |

### `txn.api.dlq`

| Config | Value |
|---|---|
| `partitions` | 3 |
| `replication.factor` | 3 |
| `retention.ms` | 604,800,000 (7 days) |
| `cleanup.policy` | `delete` |
